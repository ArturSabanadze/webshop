Dokumentation Webshop / Seminarverwaltung

Überblick

Dieses Projekt implementiert einen Webshop für Seminare mit Frontend, Backend (Login/Registrierung) und einem Admin-Dashboard zur Verwaltung von Seminaren, Terminen und Teilnehmern.

Architektur

  • Frontend: Anzeige der Seiten (Start, Shop, About, Login, Registrierung, Kontakt, Impressum, AGB).
  • Backend: Datenbankzugriff über PDO, Login-/Registrierungslogik.
  • Admin_Dashboard: Geschützter Bereich für Administratoren.

UML (Textbeschreibung)

Vereinfacht gibt es folgende Haupt-Klassen / Tabellen:

  • User (users): id, username, email, password_hash, role
  • Seminar (products): id, product_name, description, price
  • Termin (seminar_dates): id, product_id, start_datetime, end_datetime, room, max_participants
  • Teilnahme (seminar_registrations): id, user_id, seminar_date_id, registration_datetime

Fazit

Die Anforderungen der Klausur (Trennung von Frontend/Backend, PDO, Templates, Adminbereich, Seminare/Termine/Teilnehmer, serialisierte Daten) werden mit diesem Projekt erfüllt.
